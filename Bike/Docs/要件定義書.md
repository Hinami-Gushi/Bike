## 1. 背景・目的

### 1.1 背景
PBL（Project Based Learning）の海外フィールドワーク（現地での事前研修・1日目・2日目）において、参加者がそれぞれ訪れた場所・気になった場所を共有する仕組みが必要となった。LINE グループや口頭での共有では情報が流れてしまい、後から振り返ることが難しい。

### 1.2 目的
- フィールドワーク中に訪れた場所を **記録・共有** できるようにする
- グループ単位で **チームごとの観点で** 場所情報を集約する
- 地図と写真で **直感的に把握** できるようにする
- 「あえて弱いアプリ」として実装し、現地での利用を通じて **追加要件を発見** する

## 2. ターゲットユーザー

| 区分 | 想定ユーザー | ニーズ |
|---|---|---|
| 投稿者・閲覧者 | フィールドワーク参加者（学生） | 訪問場所の記録・他グループとの情報共有 |
| 閲覧者 | 引率教員 | 各グループの行動把握 |

## 3. 機能要件

### 3.1 機能一覧

| ID | 機能名 | 概要 | 認証 |
|---|---|---|---|
| F-01 | ユーザー登録 | メール・パスワード・表示名・所属グループを登録 | 不要 |
| F-02 | ログイン / ログアウト | Cookie 認証で状態を保持 | 不要 / 要 |
| F-03 | 場所一覧 | 全場所を投稿日時の降順で一覧表示。カテゴリで絞り込み可能 | 要 |
| F-04 | 場所詳細 | 1件の場所の詳細（写真・地図ピン含む）を表示 | 要 |
| F-05 | 場所投稿 | 名前・カテゴリ・住所・メモ・緯度経度・写真を投稿 | 要 |
| F-06 | 場所編集 | 同じグループのメンバーが投稿した場所を編集 | 要 |
| F-07 | 場所削除 | 同じグループのメンバーが投稿した場所を削除 | 要 |
| F-08 | 地図表示 | 投稿された場所をピンとして地図上に表示。グループごとに色分け | 要 |

### 3.2 権限ルール

- **閲覧**：ログインユーザーは全グループの場所を閲覧できる
- **投稿**：ログインユーザーは自分の所属グループとして投稿する
- **編集・削除**：投稿者と **同じグループに所属しているユーザー** のみ可能（個人単位ではなくグループ単位の権限）

### 3.3 スコープ外（v0.1 では実装しない）

- パスワードリセット / メール確認
- コメント・いいね・リアクション
- 通知機能
- 検索（カテゴリ絞り込みのみ提供）
- 多言語対応（日本語のみ）
- モバイルアプリ（Web のレスポンシブ対応のみ）
